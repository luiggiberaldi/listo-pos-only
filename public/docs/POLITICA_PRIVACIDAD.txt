1. INTRODUCCION

2. INFORMACION QUE RECOPILAMOS
--------------------------------------------------------------------------------
2.1 Datos del Negocio
- Nombre del establecimiento comercial.
- Identificador unico de hardware (HWID) del equipo.
- Configuracion de preferencias del sistema.

2.2 Datos de Transacciones
- Registro de ventas realizadas (montos, fechas, metodos de pago).
- Inventario de productos y servicios.
- Historial de operaciones del punto de venta.

2.3 Datos de Telemetria (Soporte Tecnico)
Para mejorar el servicio y brindar soporte, el Software puede recopilar:
- Version del Software instalada.
- Estado de conexion y ultima sincronizacion.
- Errores tecnicos y reportes de fallos (sin datos personales).
- Estadisticas de uso agregadas (numero de transacciones, no montos).

2.4 Datos que NO Recopilamos
El Software NO recopila ni almacena:
- Numeros de cedula de identidad de clientes finales.
- Informacion de tarjetas de credito o debito.
- Contrasenas de cuentas bancarias.
- Datos biometricos de ningun tipo.


3. USO DE LA INFORMACION
--------------------------------------------------------------------------------
3.1 Propositos Principales
Utilizamos la informacion recopilada para:
1. Operacion del Software: Permitir el funcionamiento del punto de venta.
2. Sincronizacion en la Nube: Mantener sus datos respaldados y accesibles.
3. Soporte Tecnico: Diagnosticar y resolver problemas tecnicos.
4. Mejoras del Producto: Desarrollar nuevas funcionalidades.

3.2 Lo que NO Hacemos
Nos comprometemos a NO:
- Vender sus datos a terceros.
- Compartir informacion con fines publicitarios.
- Utilizar sus datos para perfilamiento comercial.
- Acceder a sus transacciones sin su consentimiento expreso.


4. ALMACENAMIENTO Y SEGURIDAD
--------------------------------------------------------------------------------
4.1 Infraestructura en la Nube
Los datos se almacenan en servidores de Google Firebase, que cumplen con:
- Certificacion ISO 27001 (Gestion de Seguridad de la Informacion).
- Encriptacion de datos en transito (TLS 1.2+).
- Encriptacion de datos en reposo (AES-256).
- Centros de datos con seguridad fisica de nivel empresarial.

4.2 Almacenamiento Local
El Software tambien almacena datos localmente en su equipo:
- Base de datos local encriptada.
- Configuraciones de usuario protegidas.
- Cache de operaciones para funcionamiento sin conexion.

4.3 Acceso Restringido
El acceso a los datos esta protegido por:
- Sistema de autenticacion por PIN.
- Identificacion unica de hardware (HWID).
- Licencia criptografica vinculada al equipo.


5. COMPARTICION DE DATOS
--------------------------------------------------------------------------------
5.1 Con Quien Compartimos
Podemos compartir informacion unicamente con:
1. Proveedores de Infraestructura: Google Firebase (almacenamiento).
2. Autoridades Competentes: Solo ante requerimiento judicial formal.
3. Soporte Tecnico Autorizado: Con su consentimiento expreso.

5.2 Transferencias Internacionales
Los datos pueden ser procesados en servidores ubicados fuera de Venezuela 
(infraestructura de Google). Estas transferencias cumplen con estandares 
internacionales de proteccion de datos.
